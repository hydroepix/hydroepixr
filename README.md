# Model Introduction
## What is `hydroepixr`?
`hydroepixr` is an R package which supports the use of a simulation framework for modelling waterborne spread of pathogens. It is based on the DTU-DADS-AQUA model, detailed in [Romero et al. 2021](https://onlinelibrary.wiley.com/doi/abs/10.1111/tbed.14195). 

The adaptation into an R package includes major restructuring of the DTU-DADS-AQUA code, documentation of functions, and the addition of a comprehensive test suite. It also simplifies the installation of the package and streamlines it for users. 

`hydroepixr` currently supports modelling infectious salmon anemia virus (ISAv) spread at 2 levels:
- within net pen
- between net pen
The addition of between farm transmission will be implemented in a future release.

Relative to the DTU-DADS-AQUA model, there has also been a change to the SEIR model used to track within net pen infection. Animals in the DTU-DADS-AQUA model progressed linearly through the infection stages as follows: susceptible, latent, subclinical, clinical, then either dead or recovered based on a user-defined case fatality proportion. Now, animals exiting the latent stage will become either subclinically or clinically infected according to a user-defined clinically infected proportion, and then recover or die, respectively. This is referred to as the **subclinical-clinical split** See Figure 1 below for a visual depiction of this change.

Please note the package is currently in its beta version, meaning it is still being tested and there will likely be significant feature additions as development progresses. Bug fix and feature requests can be submitted on the [GitHub issue page](https://github.com/hydroepix/hydroepixr/issues).

## Naming conventions and model structure
For clarity, the following terms are defined. The **model** refers to the running of all of `hydroepixr`, through the `he_run_model` function. **Each model run can include multiple simulations**. Each of these simulations uses the same input parameters, but the stochastic elements may be different and so each simulation may have different results. This models the randomness observed in the real world. Each simulation runs over multiple days, with the infection status of animals and net pens being updated each day.

## Basic instructions to run the model
1. Install the `hydroepixr` package by running `devtools::install_github("https://github.com/hydroepix/hydroepixr")`
2. Create a model environment using `he_create_model_env()` 
3. Define the model parameters using the `he_define_*_params` family of functions, passing the model environment to the functions.
4. Load the net pen information and species information data into the model from CSV files with `he_initialize_model_env()` 
5. Run the model using the `he_run_model()` function.

## Model parameters
The parameters of the model can effectively be divided into three different groups: simulation control, spread control, and output. *Simulation control parameters* define how the simulations are run, such as the number of simulations, the maximum outbreak length, and the random seed used. *Spread control parameters* determine the beginning state of infection and how it progresses, such as the range of initially infected animals, in which infection stage initially infected animals begin, the proportion of latently infected animals which become clinically infected as opposed to subclinically infected, the scaling parameter for infection between net pens, and vaccine efficacy. *Output parameters* define how the output files for the model and each individual simulation are named and stored.

### Simulation control parameters
| Name | Default Value | Description |
|------|---------------|-------------|
| n_simulations | 10 | Number of simulations (i.e., individual simulated epidemics) to be run  |
| max_outbreak_length | 365 | Maximum length of a infection outbreak in days |
| random_seed | -10 | Random seed to use for each simulation, if negative will vary with simulation number |
| ignore_infection_status_input | FALSE | (NOT YET IMPLEMENTED boolean indicating whether or not to ignore infection status input) |
| verbose | FALSE | boolean indicating whether to generate additional output messages for debugging |

### Spread control parameters
| Name | Default Value | Description |
|------|---------------|-------------|
| n_index_infected_min | 1 | Minimum number of fish initially infected, to be calculated from a PERT distribution   |
| n_index_infected_mode | 10 | Mode of fish initially infected, to be calculated from a PERT distribution |
| n_index_infected_max | 100 | Maximum number of fish initially infected, to be calculated from a PERT distribution |
| index_net_pen_ids | NULL | Identifiers net pens to be initially infected |
| index_farm_id | NULL | Identifier of the farm to be initially infected |
| index_infection_stage | "subclinical-clinical split" | Indication of the infection stage in which the index infection will begin, valid options are "latent" and "subclinical-clinical split" |
| clinically_infected_prop | 0.89 | The proportion of animals which will enter the clinical stage upon infection, which is the same as 1 minus the proportion of animals which will enter the subclinical stage upon infection |
| net_pen_to_net_pen | 0.052 | Daily probability of infection transmission between net pens within a farm |
| vaccine_efficacy | 0 | Product of the manufacturer-reported vaccine efficacy and the population coverage of the vaccine |

### Output parameters
| Name | Default Value | Description |
|------|---------------|-------------|
| output_dir | "output" | Directory into which output should be written | 
| model_run_id | NULL | Identifier for this run of the model |
| infected_net_pen_output_file_name | "infected_net_pens.csv" | Base file name to which the run ID and simulation number will be prepended to uniquely identify a simulation's output file |

## Model outputs
Results from each simulation are written to `<output_dir>/<model_run_id>_<simulation_num>_<infected_net_pen_output_file_name>`. Each row corresponds to the state of a net pen on a given day of the simulation, with the number of animals in each infection stage indicated.
